# 实施文档

## 文件用途

本文档用于说明 CNSATimeLine 项目的目录结构、构建方式、引用配置以及部署流程。每次调整项目结构或构建脚本时，应同步更新本文档。

## 目录结构

```
e:\Projects\KSP Mod Workspace\CNSATimeLine\
├── KSP DLL\                          # KSP 1.12.5 游戏程序集目录
│   ├── Assembly-CSharp.dll           # KSP 核心脚本程序集
│   ├── Assembly-CSharp-firstpass.dll # KSP 首批编译程序集
│   ├── UnityEngine.dll               # Unity 引擎核心
│   ├── UnityEngine.CoreModule.dll    # Unity 引擎核心模块
│   └── ...                           # 其他 Unity 模块及系统库
├── readme\                           # 项目文档目录
│   ├── 开发文档.md                    # 开发目标、技术栈、约束
│   ├── 实施文档.md                    # 本文件：结构与构建说明
│   └── 功能更新.md                    # 功能变更记录
├── CNSATimeLine.csproj              # 项目文件（.NET Framework 4.7.2）
├── CNSATimeLineMod.cs               # 模组太空中心场景入口类
├── TimeLineEvent.cs                 # 时间线事件数据结构
├── TimeLineConverter.cs             # 真实日期时间与 KSP UT 转换器
├── TimeLineDataLoader.cs            # 时间线文本解析器
├── TimeLineWindow.cs                # 太空中心 IMGUI 窗口
├── 中国航天大事时间线.txt            # 时间线数据源
└── RSSTimeFormatter.dll             # 时间格式化依赖库
```

## 项目文件说明

`CNSATimeLine.csproj` 是 MSBuild 项目文件，主要配置如下：

- **目标框架**：`.NET Framework v4.7.2`
- **输出类型**：类库（`Library`）
- **输出路径**：`bin\Debug\` 或 `bin\Release\`（相对路径）
- **引用路径**：
  - `KSP DLL\` 目录下的 KSP 与 Unity 程序集
  - 项目根目录下的 `RSSTimeFormatter.dll`

## 构建方式

### 使用 MSBuild 命令行构建

```bash
# 在项目根目录执行
msbuild CNSATimeLine.csproj /p:Configuration=Release
```

### 使用 Visual Studio / Rider 构建

直接打开 `CNSATimeLine.csproj` 文件，选择 Release 配置后编译。

## 部署方式

1. 从 `bin\Release\` 目录复制编译产物 `CNSATimeLine.dll`。
2. 从项目根目录复制依赖库 `RSSTimeFormatter.dll`。
3. 从项目根目录复制数据源 `中国航天大事时间线.txt`。
4. 将上述文件放入 KSP 的 `GameData\CNSATimeLine\` 目录下：

```
KSP安装目录\GameData\CNSATimeLine\
├── CNSATimeLine.dll
├── RSSTimeFormatter.dll
└── 中国航天大事时间线.txt
```

> 说明：`CNSATimeLine.csproj` 中所有 KSP 与 Unity 程序集引用均设置 `Private=false`，
> 因此构建输出目录不会包含游戏程序集；`RSSTimeFormatter.dll` 同样设置为 `Private=false`，
> 部署时需手动从项目根目录一并复制。数据源文件需放在 DLL 同级目录，供 `TimeLineDataLoader` 读取。

## 跨平台说明

- 所有引用均使用相对路径 `$(SolutionDir)` 或直接相对项目文件路径，避免硬编码绝对路径。
- 在 Linux 或 Docker 中构建时，确保安装 `mono-complete` 或使用 .NET SDK 的 `msbuild` 工具。
