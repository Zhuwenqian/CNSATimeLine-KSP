# 开发文档

## 文件用途

本文档用于记录 CNSATimeLine 模组项目的开发目标、技术选型、核心依赖以及开发约束。每次进行功能性开发或架构调整时，应首先更新本文档，确保其与实施文档和功能更新记录保持一致。

## 项目目标

CNSATimeLine 是面向 Kerbal Space Program（KSP）1.12.5 的模组项目，用于扩展游戏内与中国航天相关的时间线、任务节点或时间格式化功能。项目以 C# 为主要开发语言，基于 .NET Framework 4.7.2 构建，依赖 KSP 游戏程序集以及 `RSSTimeFormatter.dll`。

## 技术栈

- **开发语言**：C#
- **目标框架**：.NET Framework 4.7.2（与 KSP 1.12.5 运行环境保持一致）
- **游戏平台**：Kerbal Space Program 1.12.5
- **依赖库**：
  - KSP 游戏程序集（`Assembly-CSharp.dll`、`UnityEngine.dll` 及其模块）
  - `RSSTimeFormatter.dll`（自定义时间格式化库，位于项目根目录）

## 开发约束

- 所有代码路径、引用路径均使用相对路径，确保项目在 Windows、Linux 及 Docker 环境下均可编译。
- 代码文件开头必须包含用途说明注释。
- 优先复用现有代码，避免重复实现。
- 所有可配置参数必须在注释中说明用途及调整效果。
- 调用开源项目代码时，需在注释中注明项目名称和版本。

## 核心模块

- `CNSATimeLineMod.cs`：模组入口类，负责在太空中心（SpaceCentre）场景加载时初始化模组，构建时间线事件列表并创建 UI 窗口。
- `TimeLineEvent.cs`：单条时间线事件的数据结构，包含真实日期时间、本地化键、描述（fallback）、精度标注以及映射后的 KSP UT 秒数。
- `TimeLineConverter.cs`：真实地球日期时间与 KSP UT 秒数的转换器。通过反射读取 `RSSTimeFormatter.RealDateTimeFormatter` 的私有 `epoch` 字段，实现 UT 反算；若 RSSTimeFormatter 未加载则回退到默认 epoch `1951-01-01`。
- `TimeLineDataLoader.cs`：内置中国航天大事时间线事件元数据（日期、本地化键、精度），构建 `List<TimeLineEvent>`。事件描述不再从 txt 文件读取，而是交给 KSP 的 `Localizer` 按当前游戏语言处理。
- `TimeLineWindow.cs`：太空中心场景的 IMGUI 窗口，响应 `Ctrl+L` 热键，显示事件列表并提供“加速到此时”按钮。所有可见文本均通过 `KSP.Localization.Localizer.Format` 读取。
- `Localization/*.cfg`：KSP 标准本地化文件，包含 `zh-cn`、`en-us`、`ru` 三套 UI 文本与事件描述。

## 时间映射原理

本模组依赖 `RSSTimeFormatter.dll` 提供的时间格式化器。该模组启动时会将 `KSPUtil.dateTimeFormatter` 替换为 `RealDateTimeFormatter`，其内部以自定义 `epoch`（默认 `1951-01-01`）为基准：

```
PrintDate(ut) = epoch.AddSeconds(ut)
```

反推公式：

```
ut = (targetDateTime - epoch).TotalSeconds
```

由于 `epoch` 可通过 `GameData` 配置自定义，本模组在运行时通过反射读取该字段，避免写死日期。
